1
Crear cuenta en Railway
railway.app — backend + base de datos
  • Ve a railway.app y haz clic en "Login"
  • Selecciona "Login with GitHub" y autoriza
  • Listo — ya tienes cuenta con plan gratuito
2
Crear base de datos MySQL
Railway auto-genera las credenciales
  • En Railway clic en "New Project"
  • Clic en "Add a service" → "Database" → "MySQL"
  • Espera ~1 minuto hasta que el servicio esté activo
  • Clic en el servicio MySQL → pestaña "Variables" — guarda esos valores
ℹ️ Railway genera automáticamente MYSQLHOST, MYSQLUSER, MYSQLPASSWORD, MYSQLDATABASE, MYSQLPORT. Las usaremos como referencias en el backend.
3
Cargar las tablas con TablePlus
Conectar a MySQL de Railway y ejecutar el schema

Instalar TablePlus

  • Ve a tableplus.com y descarga la versión gratuita
  • Instala y abre

Conectar a Railway

  • Clic en "Create a new connection" → selecciona MySQL
  • Llena con los datos de Railway:
Host: caboose.proxy.rlwy.net (tu MYSQL_PUBLIC_URL)
Port: 58025 (el puerto de tu URL pública)
User: root
Password: [MYSQL_ROOT_PASSWORD de Railway]
Database: railway
  • Clic en "Test" — debe salir verde ✅
  • Clic en "Connect"

Ejecutar el schema

  • Presiona Ctrl+Q para abrir el editor SQL
  • Abre el archivo database/schema.sql del proyecto
⚠️ Importante: Borrar las líneas CREATE DATABASE IF NOT EXISTS profesor_ratings y USE profesor_ratings del schema antes de ejecutar. Railway ya tiene su propia base de datos llamada railway.
  • Ejecutar cada CREATE TABLE por separado con "Run Current"
  • Verificar que aparezcan las 7 tablas en el panel izquierdo
✅ Tablas esperadas: usuarios, catedraticos, cursos, curso_catedratico, publicaciones, comentarios, cursos_aprobados
4
Deploy del Backend en Railway
Express.js conectado a MySQL

Agregar el servicio

  • En el mismo proyecto Railway clic en "Add a service"
  • Selecciona "GitHub Repo"
  • Busca y selecciona tu repositorio
ℹ️ Si el repo no aparece, clic en "Configure GitHub App" y da acceso al repositorio desde GitHub.

Configurar Root Directory

  • Ve a Settings del servicio
  • En "Root Directory" escribe:
profesor-ratings-app/backend
⚠️ La ruta depende de cómo quedó el ZIP al subirse. Verificar la estructura en GitHub antes de escribir la ruta.

Variables de entorno

Ve a la pestaña Variables y agrega usando "Add Reference" para las de MySQL:

Variable Valor Tipo
DB_HOST ${{MySQL.MYSQLHOST}} Referencia
DB_USER ${{MySQL.MYSQLUSER}} Referencia
DB_PASSWORD ${{MySQL.MYSQLPASSWORD}} Referencia
DB_NAME ${{MySQL.MYSQLDATABASE}} Referencia
DB_PORT ${{MySQL.MYSQLPORT}} Referencia
PORT 5000 Texto
NODE_ENV production Texto
JWT_SECRET jwt_secure_key_usac_2026 Texto
JWT_EXPIRE 7d Texto
CORS_ORIGIN * Texto

Generar dominio público

  • Ve a Settings → Networking
  • Clic en "Generate Domain"
  • Copia la URL generada — la necesitarás para el frontend
✅ Verificar que el backend esté activo accediendo a: https://tu-dominio.railway.app/api/health — debe responder con JSON.
5
Ajuste en el código del frontend
Cambio obligatorio antes del deploy
⚠️ Este paso es crítico. El archivo frontend/src/services/api.js tiene la URL hardcodeada como localhost. Hay que cambiarlo para que use la variable de entorno.
frontend/src/services/api.js — ANTES ❌ const API_URL = 'http://localhost:5000/api';
frontend/src/services/api.js — DESPUÉS ✅ const API_URL = process.env.REACT_APP_API_URL || 'http://localhost:5000/api';
  • Hacer el cambio en VS Code
  • Commit y push a GitHub (con GitKraken o terminal)
6
Deploy del Frontend en Vercel
React.js — deploy automático desde GitHub
  • Ve a vercel.com → "Sign Up" → "Continue with GitHub"
  • Clic en "Add New Project" → busca el repo
  • En "Root Directory" escribe: profesor-ratings-app/frontend
  • En "Environment Variables" agrega:
REACT_APP_API_URL=https://tu-dominio.railway.app/api
  • Clic en "Deploy" — Vercel construye y despliega automáticamente
  • Esperar que el status diga "Ready"
!
Errores comunes y soluciones
Problemas que ocurrieron durante el deploy
Failed to open the referenced table 'cursos'
❌ Causa: El schema incluye CREATE DATABASE y USE profesor_ratings al inicio.
✅ Solución: Borrar esas dos líneas del schema antes de ejecutar en TablePlus. Railway ya usa su propia base de datos "railway".
Query syntax error near '' at line 1
❌ Causa: TablePlus no ejecuta múltiples queries separadas por punto y coma en un solo run.
✅ Solución: Ejecutar cada CREATE TABLE por separado con "Run Current".
ERR_CONNECTION_REFUSED — localhost:5000
❌ Causa: El archivo api.js tiene la URL hardcodeada como localhost en lugar de usar la variable de entorno.
✅ Solución: Cambiar a process.env.REACT_APP_API_URL || 'http://localhost:5000/api' y hacer push.
ENOTFOUND MYSQLHOST
❌ Causa: Las variables DB_HOST, DB_USER, etc. se guardaron como texto plano "MYSQLHOST" en lugar de referencias a las variables de MySQL.
✅ Solución: En Railway Variables, usar el ícono {} → "Add Reference" → seleccionar ${{MySQL.MYSQLHOST}} etc.
No repositories found en Railway
❌ Causa: El repo es de otro usuario y Railway no tiene acceso.
✅ Solución: Descargar el ZIP del repo original, subirlo a un repo propio en GitHub, y conectar ese repo a Railway.
There was an error deploying from source
❌ Causa: El Root Directory está mal configurado o vacío.
✅ Solución: Verificar la estructura del repo en GitHub y configurar Root Directory como "profesor-ratings-app/backend".
